# 微信公众号合规检查 Skill

> 对照微信公众平台运营规范，逐条检查文章合规风险。只做检查，不改文章。

## 这是什么

一个 Claude Code Skill，检查公众号文章是否触碰微信官方 9 条红线。覆盖 9 项违规类型 + 补充条款（诱导引流、滥用原创、AIGC 标注等），每项违规引用官方规范原文。

**适用场景：**
- 文章发之前"过一遍"
- 检查配图合规、AIGC 标记、历史文章同质化
- 涉及健康/心理/历史等敏感主题时加保护声明

## 四类专项检查

| 专项 | 覆盖场景 |
|---|---|
| 心理健康/医疗 | 红线 5.1 — 强制标注"不替代专业诊断" |
| 历史/传统文化 | 红线 5.2 — 强制标注"仅供阅读参考" |
| 标准/法规引用 | 红线 8 — 检查引用版本是否现行有效 |
| 多图配文 | 截图是否脱敏、AIGC 图是否冒充证据 |

## 安装

```bash
# 将整个文件夹放入 Claude Code 的 skills 目录
```

或通过 npm：
```bash
npx skills add yanwuyou/wechat-compliance-checker-skill
```

## 使用

```
检查我最新这篇公众号文章的合规性
```

Skill 会自动加载规范原文，逐条审查，输出结构化报告：

```
# 合规检查报告
- 通过项：x/9
- 违规项：x 条
- 综合判断：可发布 / 修改后发布 / 不建议发布
```

## 规范来源

- `references/platform-rules.md` — 微信公众平台推荐运营规范 9 条红线
- `references/platform-rules-basic.md` — 基础运营规范补充条款
- `references/platform-cases.md` — 官方低创作度案例解析

## 边界

- 只做检查，不改文章
- 报告是参考性的，不是平台最终判决
- 需要用户提供文章全文（含标题、正文、配图描述）
